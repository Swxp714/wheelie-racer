# WHEELIE RACER - Game Design Document (GDD)

> **장르**: 모션컨트롤 아케이드 레이싱
> **플랫폼**: PC (웹캠 필수)
> **엔진**: Python Ursina Engine
> **입력방식**: MediaPipe 손 인식 (행동인식)
> **아트 스타일**: Smashy Road / Crossy Road 복셀 스타일
> **작성일**: 2026-04-09

## 1. 게임 컨셉

### 1.1 한줄 요약
**"양손으로 바이크를 조종하고, 윌리로 속도를 높이되, 너무 높이 들면 뒤로 꼬꾸라져 사망하는 모션컨트롤 레이싱 게임"**

### 1.2 핵심 재미 요소
| 요소 | 설명 |
|------|------|
| **윌리 리스크/리워드** | 손을 높이 들수록 속도↑, but 과하면 사망 → 긴장감 |
| **직관적 조작** | 실제 핸들 잡듯 양손으로 조종 → 몰입감 극대화 |
| **도발 시스템** | 뻐큐 제스처로 도발 → 웃음 유발, 소셜 콘텐츠 |
| **복셀 아트** | Smashy Road 스타일 → 가볍고 재밌는 비주얼 |

### 1.3 게임 루프
```
시작 → 레이싱 → 윌리로 가속 → 장애물 회피 → 
  ├─ 균형 유지 성공 → 더 빠른 속도 → 더 높은 점수
  └─ 균형 실패 → 뒤로 넘어짐 → 사망 → 리스폰 → 반복
```

## 2. 조작 시스템 (MediaPipe 손 인식)

### 2.1 제스처 매핑

| 제스처 | 감지 방식 | 게임 액션 |
|--------|-----------|-----------|
| **양손 높이 올림** | 양손 wrist(0) Y좌표 상승 | 앞바퀴 들기 (윌리) → 속도 증가 |
| **양손 높이 내림** | 양손 wrist(0) Y좌표 하강 | 앞바퀴 내리기 → 안정 주행 |
| **양손 좌로 기울임** | 양손 평균 기울기 angle < 0 | 좌회전 |
| **양손 우로 기울임** | 양손 평균 기울기 angle > 0 | 우회전 |
| **양손 주먹** | 모든 손가락 접힘 (tip < pip) | 가속 |
| **양손 펼침** | 모든 손가락 펼침 | 브레이크 |
| **오른손 뻐큐** | 중지만 펴짐 + 나머지 접힘 | 도발 (화면 이펙트 + 소리) |

### 2.2 MediaPipe 랜드마크 활용

```
손 랜드마크 21개 포인트:
  0: WRIST (손목) → Y좌표로 손 높이 감지
  8: INDEX_FINGER_TIP → 손가락 펴짐 감지
  12: MIDDLE_FINGER_TIP → 뻐큐 감지의 핵심
  9: MIDDLE_FINGER_MCP → wrist(0)과의 각도로 기울기 계산
  
기울기 계산:
  angle = atan2(landmark[9].y - landmark[0].y, landmark[9].x - landmark[0].x)
  steering = angle 기반 좌우 회전값 (-1.0 ~ 1.0)

윌리 높이:
  wheelie_amount = 1.0 - wrist.y  (Y축 반전, 높을수록 값 큼)
  위험 구간: wheelie_amount > 0.8 → 3초 후 사망
  최적 구간: wheelie_amount 0.4~0.7 → 최대 속도 보너스
```

### 2.3 뻐큐 감지 로직
```python
def is_middle_finger(landmarks):
    """중지만 펴지고 나머지는 접힌 상태 감지"""
    middle_up = landmarks[12].y < landmarks[10].y   # 중지 TIP < PIP
    index_down = landmarks[8].y > landmarks[6].y    # 검지 접힘
    ring_down = landmarks[16].y > landmarks[14].y   # 약지 접힘
    pinky_down = landmarks[20].y > landmarks[18].y  # 새끼 접힘
    return middle_up and index_down and ring_down and pinky_down
```

## 3. 게임 메카닉

### 3.1 윌리 시스템 (핵심 메카닉)

```
윌리 각도 (0° ~ 90°):
  
  0°  ████████████████  안전 구간 - 기본 주행
  20° ████████████████  가속 시작 - 속도 1.2x
  40° ████████████████  최적 구간 - 속도 1.5x ⭐
  60° ██████████░░░░░░  위험 구간 - 속도 2.0x ⚠️ 흔들림 시작
  75° █████░░░░░░░░░░░  극위험 - 속도 2.5x 🔴 심한 흔들림
  90° ░░░░░░░░░░░░░░░░  사망! 뒤로 넘어짐 💀
```

**윌리 물리 파라미터:**
| 파라미터 | 값 | 설명 |
|----------|-----|------|
| `base_speed` | 15 m/s | 기본 주행 속도 |
| `wheelie_speed_mult` | 1.0 ~ 2.5x | 윌리 각도에 비례 |
| `gravity_torque` | 9.81 | 아래로 당기는 힘 |
| `balance_wobble` | 60° 이상 시작 | 랜덤 흔들림 증가 |
| `death_angle` | 85° ~ 90° | 이 각도 초과 시 사망 |
| `recovery_speed` | 3.0/s | 윌리 해제 시 복귀 속도 |

### 3.2 스티어링 시스템
```
양손 기울기 → 좌우 회전
  - 최대 회전 각도: ±35°
  - 회전 반응 속도: lerp(current, target, dt * 5)
  - 속도에 따른 회전 감소: 고속일수록 회전 반경 증가
```

### 3.3 사망 & 리스폰
```
사망 조건:
  1. 윌리 각도 90° 초과 → 뒤로 넘어짐 애니메이션
  2. 장애물 충돌 → 폭발 이펙트
  3. 트랙 이탈 → 추락

리스폰:
  - 2초 대기 (사망 애니메이션)
  - 마지막 체크포인트에서 리스폰
  - 속도 0에서 재시작
  - 점수는 유지 (하이스코어 기록)
```

### 3.4 도발 시스템 (뻐큐)
```
오른손 뻐큐 감지 시:
  - 화면에 "🖕" 이모지 이펙트 팝업
  - 뻐큐 사운드 이펙트 재생
  - 바이크 위에 말풍선 표시
  - NPC가 있으면 NPC 반응 (놀람/화남)
  - 게임플레이에 영향 없음 (순수 도발)
  - 쿨다운: 3초
```

## 4. 월드 & 트랙 디자인

### 4.1 트랙 구성
```
[직선 구간] → [완만한 커브] → [장애물 구간] → [점프 램프] → 
[S자 구간] → [윌리 보너스 구간] → [반복...]
```

### 4.2 장애물 유형
| 장애물 | 비주얼 | 행동 |
|--------|--------|------|
| 차량 | 복셀 자동차 | 도로에 정지/저속 이동 |
| 콘 | 주황색 삼각콘 | 고정 위치 |
| 바리케이드 | 줄무늬 차단봉 | 고정, 점프로만 회피 |
| 유턴 차량 | 갑자기 유턴하는 차 | 예측 불가 이동 |
| 보행자 | 복셀 사람 | 횡단보도 횡단 |

### 4.3 아이템 (도로 위)
| 아이템 | 효과 |
|--------|------|
| 🔧 수리킷 | 흔들림 감소 (일시적 안정성 부여) |
| ⭐ 스타 | 점수 x2 (10초) |
| ⚡ 부스트 | 일시적 최대 속도 증가 |
| 🛡️ 쉴드 | 충돌 1회 무효 |

### 4.4 환경 (Smashy Road / Crossy Road 스타일)
```
도시 구간:
  - 복셀 건물, 가로등, 횡단보도
  - 신호등 (장식용)
  - NPC 차량 트래픽

교외 구간:
  - 나무, 풀밭, 울타리
  - 동물 (닭, 소 - Crossy Road 오마쥬)
  - 좁은 시골길

고속도로 구간:
  - 넓은 도로, 빠른 트래픽
  - 분리대, 터널
  - 최고 속도 구간
```

## 5. 아트 디자인 시스템

### 5.1 비주얼 스타일
```
참고 게임: Smashy Road + Crossy Road
스타일: Vector Voxel Art (90도 각도만 사용, 깔끔한 복셀)
도구: MagicaVoxel → OBJ Export → Ursina Engine
```

### 5.2 카메라
```
프로젝션: 직교(Orthographic) 또는 약간의 원근
Y축 회전: 45° (아이소메트릭)
X축 기울기: 30~35° (위에서 살짝 내려다보는 시점)
바이크 뒤쪽 팔로우 카메라
윌리 시: 카메라 살짝 줌아웃 + 화면 흔들림
```

### 5.3 컬러 팔레트
```
도로:       #888888 (회색)
잔디:       #4CAF50 (초록)
하늘:       #85D2FE (하늘색)
바이크:     #F95858 (빨강) / #E0834E (주황)
경고:       #FF4444 (적색)
UI 텍스트:  #FFFFFF (흰색)
그림자:     #333333 (어두운 회색)
부스트:     #FFD700 (금색)
```

### 5.4 셰이딩
```
- 버텍스 컬러 (UV 매핑 없음, 텍스처 없음)
- 플랫 셰이딩 (면 단위 노말)
- 단일 방향광 (태양)
- 모델 하단에 어두운 색상으로 가짜 앰비언트 오클루전
- 캐릭터/차량 아래 원형 그림자 블롭
```

### 5.5 이펙트
| 이펙트 | 상황 | 구현 |
|--------|------|------|
| 바퀴 연기 | 윌리 중 | 파티클 시스템 (뒷바퀴) |
| 속도 라인 | 고속 주행 | 화면 양쪽 스트림 라인 |
| 화면 흔들림 | 위험 구간/충돌 | 카메라 shake |
| 폭발 | 사망 시 | 복셀 파편 파티클 |
| 뻐큐 이펙트 | 도발 시 | 말풍선 + 이모지 팝업 |
| 부스트 잔상 | 윌리 가속 | 바이크 뒤 모션 블러 |
